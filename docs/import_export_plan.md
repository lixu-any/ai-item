# Nixu 导入 / 导出功能规划

## 目标

允许用户将 Nixu 的配置数据（主机、分组、命令片段等）导出为文件，并在另一台机器或重装后导入恢复。

---

## 导出内容

| 数据类型 | 是否导出 | 备注 |
|----------|----------|------|
| 主机列表（hosts） | ✅ | 包含 IP、端口、用户名 |
| 分组（groups） | ✅ | 包含名称、颜色 |
| 命令片段（snippets） | ✅ | |
| 应用设置（settings） | ✅ 可选 | 字体、主题等，不含 AI Key |
| 凭据（credentials） | ⚠️ 可选 | 密码/密钥**明文**，用户需知情确认 |

---

## 文件格式

扩展名：`.nixu`（本质是 JSON）

```json
{
  "version": "1",
  "exported_at": "2026-03-16T14:46:00Z",
  "app": "Nixu",
  "groups": [...],
  "hosts": [...],
  "snippets": [...],
  "settings": {...},
  "credentials": [...]
}
```

---

## 实现方案

### 后端（Rust） — 新增 2 个命令

#### `export_data`
1. 查询 DB 中全部 hosts / groups / snippets / settings / credentials（可选）
2. 序列化为 JSON
3. 通过 `dialog::save_file` 让用户选择保存路径
4. 写入磁盘

#### `import_data`
1. 用 `dialog::open_file` 让用户选择 `.nixu` 文件
2. 反序列化 JSON，校验 `version` 字段
3. 冲突策略：**跳过重复**（默认）或**覆盖**（用户确认）
4. 批量插入 DB

### 前端（Vue） — UI 入口

**方案 A**：放在设置页 → 新增「数据管理」Tab（推荐）

**方案 B**：主界面顶部菜单（后续再加）

设置「数据管理」Tab 内容：
- 导出按钮（含「是否包含凭据」勾选项）
- 导入按钮（选择文件后显示预览：X 个主机、Y 个分组...）
- 导入完成后 toast 提示结果

---

## 安全说明

> ⚠️ 凭据（密码/私钥）以**明文**存储在导出文件中。
> 建议用户妥善保管 `.nixu` 文件，不要上传至公开位置。
> 后续版本可考虑加密导出（AES-256 + 用户密码）。
