# AI-Term-Shell 开发步骤文档 (Implementation Roadmap)

本文档将开发过程划分为 6 个核心阶段，建议按顺序逐步推进。

## 阶段 1：项目骨架与基础环境 (第 1-2 周)
- [x] **1.1 环境初始化**: 
    - 使用 `pnpm create tauri-app` 创建项目 (Tauri v2 + Vue/React)。
    - 配置 `Cargo.toml`，引入 `rusqlite (bundled)`, `tokio`, `serde`, `ssh2` (或 `russh`)。
- [x] **1.2 数据库基座**: 
    - 编写 Rust 侧的数据库初始化逻辑（创建 `hosts`, `groups` 等表）。
    - 实现第一个 `invoke` 命令：从 SQLite 读取并返回服务器列表。
- [ ] **1.3 安全存储原型**: 
    - 集成 `keyring-rs`，实现对一个测试密码的存取。

## 阶段 2：终端核心 (SSH & xterm.js) (第 3-4 周)
- [ ] **2.1 前端终端渲染**: 
    - 在前端集成 `xterm.js`，并对接 `xterm-addon-fit`。
- [ ] **2.2 后端 SSH 通道**: 
    - 实现 Rust 侧的异步 SSH 连接逻辑。
    - 建立前端输入 -> IPC -> Rust (SSH Stdin) 的链路。
    - 建立 Rust (SSH Stdout) -> Event -> 前端显示 的回传链路。
- [ ] **2.3 窗口自适应**: 
    - 处理终端窗口 Resize 事件，确保远程控制台行数列数同步。

## 阶段 3：AI 增强功能 - 第一部分 (自然语言) (第 5-6 周)
- [ ] **3.1 AI 适配层**: 
    - 编写 AI 请求客户端（支持 DeepSeek 流式返回）。
    - 制定系统 Prompt（专门针对命令生成的提示词）。
- [ ] **3.2 智能命令框**: 
    - 前端增加 `Alt+Space` 弹出 AI 搜索框。
    - 实现“自然语言 -> 生成命令 -> 点击执行”的闭环流程。

## 阶段 4：UI 经营与 SFTP (第 7-8 周)
- [ ] **4.1 侧边栏与会话管理**: 
    - 实现美观的服务器列表 UI（支持拖拽排序、分组）。
- [ ] **4.2 SFTP 文件浏览器**: 
    - 实现基础的文件列表读取（`ls -la` 的结构化解析或 SFTP 协议）。
    - 实现文件的上传、下载、重命名逻辑。
- [ ] **4.3 可视化监控**: 
    - 实现简易的监控面板（通过 SSH 运行 `top` 或 `free -m` 并解析）。

## 阶段 5：AI 增强功能 - 第二部分 (诊断与解释) (第 9-10 周)
- [ ] **5.1 错误自动诊断**: 
    - 后端监听 `stderr` 输出，当检测到关键词或非零状态码时，触发 AI 分析。
- [ ] **5.2 划词解释**: 
    - 支持在终端中选中文本后，弹出 AI 菜单进行“命令解释”。

## 阶段 6：优化、打包与发布 (第 11-12 周)
- [ ] **6.1 主题与美化**: 
    - 适配 macOS 毛玻璃效果，自定义终端配色。
- [ ] **6.2 跨平台构建**: 
    - 配置 GitHub Actions 实现自动构建 `dmg`, `msi`, `deb` 安装包。
- [ ] **6.3 安全审计**: 
    - 最终检查本地加密存储和网络请求的安全性。
