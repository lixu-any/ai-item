## 1. 第一次启动

启动 Nixu Terminal 后，你会看到主界面：

- **左侧**：主机侧边栏（初始为空）
- **右侧**：终端区域（未连接时显示"未连接"）
- **顶部**：标签栏 + 工具栏

**推荐第一步**：先完成 [AI 配置](#2-配置-ai)，再添加你的服务器。

---

## 2. 配置 AI

Nixu 的大部分 AI 功能（时光机纪要、全息搜索、AI 诊断）都需要接入外部大模型 API。

### 操作步骤

1. 点击左侧底部的 **⚙️ 设置** 图标，打开设置面板
2. 切换到 **AI 配置** 选项卡
3. 填写以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| API 地址 | 兼容 OpenAI 格式的服务地址 | `https://api.deepseek.com/v1` |
| API Key | 对应服务商的访问密钥 | `sk-xxxxxxxx` |
| 模型名称 | 使用的具体模型 | `deepseek-chat` |

4. 点击 **保存**，AI 功能即刻生效

### 推荐服务商

- **DeepSeek**（推荐）：性价比最高，国内直连，适合高频使用
- **硅基流动**：支持多种开源模型，价格实惠
- **OpenAI**：效果最佳，需要代理访问
- **本地 Ollama**：完全离线，地址填 `http://localhost:11434/v1`，Key 随便填

## 3. 主机管理

### 添加主机

1. 点击侧边栏顶部的 **+ 添加主机** 按钮
2. 填写连接信息：
   - **名称**：自定义显示名称（如"腾讯云生产"）
   - **主机/IP**：服务器地址
   - **端口**：默认 22
   - **用户名**：登录用户（通常是 `root` 或 `ubuntu`）
   - **认证方式**：
     - **密码认证**：直接填入密码
     - **私钥认证**：粘贴私钥内容，或填写本地私钥文件路径
3. 点击 **保存**

### 主机分组

1. 点击 **+ 添加分组**，创建分组（如"生产环境"、"测试环境"）
2. 添加主机时在 **所属分组** 下拉框中选择
3. 已有主机可通过**拖拽**到目标分组进行移动

### 凭据中心

如果多台服务器共用相同的账号密码，可使用凭据中心统一管理：

1. 点击左下角 **🔑 凭据中心**
2. 添加凭据（名称 + 用户名 + 密码/私钥）
3. 在添加/编辑主机表单中点击 **选择凭据** 一键填入

### 删除与编辑主机

- 在主机列表中，鼠标悬停到主机名旁会出现 **编辑 ✏️** 和 **删除 🗑️** 图标
- 点击对应图标执行操作

## 4. 连接服务器

### SSH 连接

- 在侧边栏**单击主机名**即可发起 SSH 连接
- 连接成功后，会在顶部标签栏新建一个标签页
- **最近连接**区域会显示最近 3 台连接过的服务器，方便快速重连

### 本地终端

- 点击侧边栏底部的 **本地终端** 按钮，打开一个本机 Shell 会话

### 多标签管理

- 点击不同标签页切换会话
- 每个标签页右上角可切换 **终端视图** / **文件视图（SFTP）**
- 右键点击标签页，可选择：
  - **复制标签**：再开一个相同主机的连接
  - **重新连接**：断线重连
  - **断开连接**：断开并关闭标签
  - **关闭 / 关闭其他 / 关闭全部**

## 5. 终端基本操作

### 终端内快捷键

| 操作 | 快捷键（Mac） |
|------|--------------|
| 复制选中文本 | `Cmd + C` |
| 粘贴 | `Cmd + V` |
| 终端内搜索 | `Cmd + F` |
| 清屏 | 右键菜单 → 清屏 |
| 全息语义搜索 | `Cmd + Shift + R` |

### 自动补全

输入命令时，Nixu 会根据历史记录在光标后以**灰色幽灵文字**显示补全候选：

- `Tab`：接受补全
- 继续输入：忽略补全，自定义内容

> 可在设置中开启/关闭自动补全功能。

### 右键菜单

在终端内右键单击，可访问：
- 复制 / 粘贴
- 终端内搜索
- 清屏

## 6. AI 功能使用指南

### 🕐 时光机纪要（Session Recap）

**无需任何操作**，每次重新连接已有操作历史的服务器时，AI 会自动在终端**左上角**弹出一行简报：

```
✨ 检测到您上个栈周期主体为修改 Nginx 配置并重启 Docker daemon。
```

简报会在约 10 秒后自动消散，不干扰终端操作。

> **首次连接**某台服务器时不会显示（因为还没有历史记录）。历史记录会在你每次执行命令后自动累积。

---

### 🔍 全息语义搜索

**快捷键**：`Cmd + Shift + R`

1. 按下快捷键，屏幕中央弹出聚光灯搜索框
2. 用**自然语言**输入你想找的命令，例如：
   - `"查看磁盘空间的命令"`
   - `"上次我是怎么重启 nginx 的？"`
3. 按 `Enter`，AI 在该服务器的历史命令中进行语义检索
4. 找到的命令会**自动写入终端光标位置**，按 `Enter` 执行

> 全息搜索的历史数据**按服务器隔离**，互不干扰。

---

### 🛡️ 高危命令拦截

当你输入被识别为危险的命令时（如 `rm -rf /`、`chmod 777 /etc`），会弹出**安全结界**：

1. 界面显示被拦截的命令内容
2. 出现一道**数学验证题**（例如：`7 × 8 = ?`）
3. 你必须在输入框中**输入正确答案**，才能放行执行

> 如果不确定是否继续，点击 **取消** 按钮直接撤销命令。

---

### 🤖 AI 诊断侧边栏

当终端出现报错时，可以让 AI 帮你分析：

**方式一（手动）：**
1. 点击左侧工具栏的 **🤖 AI** 图标打开侧边栏
2. 选中报错文字，粘贴到对话框，发送给 AI

**方式二（自动触发）：**
- 未来版本将支持识别到 Error 关键词时自动呼叫 AI

侧边栏的 AI 回复支持：
- **一键复制**代码块
- **注入终端**（代码块中有命令时）直接写入当前会话

---

### 📊 NanoHUD 状态图

连接 SSH 后，终端**右上角**会悬浮一个极简状态面板（如果在设置中开启）：

- 实时显示目标服务器的 CPU 使用率和内存使用率
- 带迷你折线历史图，5 秒自动刷新
- 鼠标点击不会被它拦截（透传到终端）

> 可在 **设置 → 终端** 中开启/关闭。

## 7. 文件操作（SFTP）

每个 SSH 标签页都内置了文件浏览器，无需额外配置。

### 切换文件视图

点击标签页右上角的 **文件** 按钮，即可切换到 SFTP 文件管理视图。

### 上传文件（rz 触发）

在终端中执行 `rz` 命令后，Nixu 自动弹出系统文件选择框，选好文件后自动上传到当前目录。

### 下载文件（sz 触发）

在终端中执行 `sz <文件名>` 后，Nixu 自动弹出系统保存框，选好路径后自动下载。

### SFTP 文件浏览器

在文件视图中：
- **双击目录**进入
- **点击文件**可查看或下载
- 支持**拖拽本地文件**到浏览器窗口进行上传

## 8. 命令片段库

把你常用的长命令保存为片段，随时一键执行。

### 添加片段

1. 点击左侧工具栏 **{ }** 图标打开片段侧边栏
2. 点击 **+ 新建片段**
3. 填写名称和命令内容，保存

### 使用片段

- 在片段列表中点击某条片段的 **▶ 执行**，命令会直接写入当前终端并执行

### Spotlight 搜索片段

按 `Cmd + K` 唤出全局搜索，输入片段名称，回车即可执行。

## 9. 会话录制与回放

### 开始录制

点击标签页上的 **⏺ 录制** 按钮，开始录制终端操作（按钮变红并显示 `⏹ REC`）。

### 停止录制

再次点击 **⏹ REC** 按钮，会弹出文件保存对话框，选择路径保存为 `.cast` 文件（Asciicast 格式）。

### 回放录像

1. 点击左侧侧边栏的 **🎬 打开录像** 图标
2. 选择 `.cast` 文件
3. Nixu 在新标签页中以内置播放器回放，支持进度控制

## 10. 广播模式

广播模式允许你**同时向多个终端会话发送相同命令**，适合批量运维场景。

### 开启广播

点击顶部工具栏的 **📡 广播** 图标，进入广播模式（顶部栏变色提示）。

### 选择目标

- 进入广播模式后，每个标签页上会出现一个 **复选框**
- 勾选你想要接收广播的标签页（不选则默认全部标签页）

### 发送广播

在底部输入框中输入命令，按 `Enter` 同时向所有选中的会话发送。

### 退出广播

再次点击工具栏的广播图标，退出广播模式。

## 12. 个性化设置

点击左下角 **⚙️ 设置** 进入设置面板：

### 终端外观

| 设置项 | 说明 |
|--------|------|
| 字体 | 从系统字体列表中选择，推荐 JetBrains Mono / Fira Code |
| 字号 | 建议 13–16px |
| 行高 | 建议 1.2–1.5 |
| 光标样式 | 块状 / 下划线 / 竖线 |
| 渲染引擎 | WebGL（性能更好）/ DOM（兼容性更好）|

### AI 功能

| 设置项 | 说明 |
|--------|------|
| NanoHUD | 开启/关闭右上角的实时状态图 |
| 自动补全 | 开启/关闭幽灵文字补全提示 |
| API 配置 | 填写 AI 服务地址、Key、模型名 |

### SSH 连接

| 设置项 | 说明 |
|--------|------|
| 连接超时 | 默认 15 秒 |
| Keepalive | 默认 60 秒，防止 SSH 因空闲断开 |
| 标签页标题模式 | 显示主机 IP 或主机名称 |

### 数据管理

| 功能 | 说明 |
|------|------|
| 导出主机配置 | 将所有主机信息导出为 JSON 文件备份 |
| 导入主机配置 | 从 JSON 文件批量导入主机配置 |

## 常见问题

**Q: 时光机纪要为什么不显示？**  
A: 需要先执行几条命令，系统积累历史后才会在下次连接时显示。另外请确认 AI API 配置正确。

**Q: 全息搜索一直在加载怎么办？**  
A: 检查设置中的 AI API 地址和 Key 是否填写正确，并确认网络可以访问对应服务。

**Q: 高危命令拦截误判了一条正常命令？**  
A: 通过数学验证后即可正常执行。拦截规则基于关键词匹配，未来版本会优化精度。

**Q: NanoHUD 的数据不刷新？**  
A: NanoHUD 通过 SSH 执行轻量探测命令获取数据，需要服务器具备基本的 shell 环境（通常都有）。如果仍然无数据，尝试断开重连。

**Q: 连接服务器失败？**  
A: 依次检查：① IP 和端口是否正确 ② 防火墙/安全组是否放行 22 端口 ③ 用户名和密码/私钥是否匹配 ④ 设置中增大连接超时时间。
