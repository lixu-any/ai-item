## 二、AI 功能数据流

```mermaid
sequenceDiagram
    participant U as 用户
    participant T as Terminal.vue
    participant IPC as Tauri IPC
    participant DB as db.rs (SQLite)
    participant AI as ai.rs
    participant API as AI API

    Note over T,DB: 连接服务器时触发时光机

    T->>IPC: get_recent_session_logs(hostId, limit=30)
    IPC->>DB: 查询 session_logs 表
    DB-->>IPC: 返回命令历史列表
    IPC-->>T: logs[]

    alt 有历史记录
        T->>IPC: ask_ai(prompt, systemPrompt)
        IPC->>AI: 构建请求
        AI->>API: POST /chat/completions
        API-->>AI: AI 生成摘要
        AI-->>IPC: 返回文本
        IPC-->>T: summary string
        T->>T: recapMessage = summary\n显示左上角浮层
    end

    Note over T,DB: 用户每次执行命令后记录

    U->>T: 输入命令 + 回车
    T->>T: 检测提示符变化\n提取执行的命令
    T->>IPC: log_terminal_command(hostId, command)
    IPC->>DB: INSERT INTO session_logs

    Note over T,API: 全息语义搜索

    U->>T: Cmd+Shift+R
    T->>T: 显示全息搜索弹窗
    U->>T: 输入自然语言查询
    T->>IPC: get_recent_session_logs(hostId)
    IPC->>DB: 查询历史
    DB-->>T: 命令列表
    T->>IPC: ask_ai(构建检索 prompt)
    IPC->>AI: 语义匹配
    AI->>API: POST /chat/completions
    API-->>T: 最匹配的命令
    T->>T: 注入到终端光标位置
```

## 三、SSH 会话生命周期

```mermaid
stateDiagram-v2
    [*] --> 初始化: 用户点击主机

    初始化 --> 建立SSH连接: invoke open_ssh_session
    建立SSH连接 --> 连接失败: 超时/认证失败
    连接失败 --> [*]: closeSession()

    建立SSH连接 --> 会话就绪: 连接成功
    会话就绪 --> 数据流转: SSE 事件监听
    
    state 数据流转 {
        [*] --> 接收数据: sse-data-{sessionId}
        接收数据 --> Prompt检测: 分析字节流
        Prompt检测 --> 命令提取: 检测到提示符
        命令提取 --> 写入DB: log_terminal_command
        写入DB --> 接收数据
    }

    数据流转 --> 会话就绪: 继续等待

    会话就绪 --> 安全检测: 用户回车
    安全检测 --> 高危拦截: 匹配危险命令
    高危拦截 --> 放行: 通过数学验证
    高危拦截 --> 取消: 用户点击取消
    放行 --> 发送命令: write_to_ssh
    会话就绪 --> 发送命令: 普通命令
    发送命令 --> 数据流转

    会话就绪 --> 断开连接: 用户关闭标签
    断开连接 --> [*]: close_ssh_session\ncloseSession()
```

## 四、数据库表结构

```mermaid
erDiagram
    hosts {
        integer id PK
        text name
        text host
        integer port
        text username
        text password
        text private_key
        integer group_id FK
    }

    groups {
        integer id PK
        text name
        integer parent_id FK
    }

    session_logs {
        integer id PK
        integer host_id FK
        text command
        text session_id
        datetime created_at
    }

    credentials {
        integer id PK
        text name
        text username
        text password
        text private_key
    }

    settings {
        text key PK
        text value
    }

    recents {
        integer id PK
        integer host_id FK
        datetime connected_at
    }

    hosts ||--o{ session_logs : "has"
    hosts ||--o{ recents : "recorded in"
    hosts }o--|| groups : "belongs to"
    groups }o--o| groups : "parent_id (nested)"
```
